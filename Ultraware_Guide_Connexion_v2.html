Guide de connexion
Ultraware — Ultra3000

Accéder au drive 2098-DSD-075 et lire sa configuration de base

Prérequis matériels PC Windows + Ultraware v1.8+
Cable RS-232 DB9 → CN3 du drive
Drive sous tension (LED verte allumée)
Ref. logiciel : 2098-UWCPRG
Phase A — Installation et câblage Phase B — Connexion au drive Phase C — Lecture et sauvegarde Phase D — Reprogrammer le drive pour un autre moteur
Phase A — Installation et câblage Logiciel + connexion physique PC → drive
A1
Installer le logiciel Ultraware
Référence Allen-Bradley 2098-UWCPRG — version 1.8 ou supérieure
1Télécharger Ultraware depuis le site Rockwell Automation (ab.com) ou depuis le CD fourni avec le drive (réf. 2098-UWCPRG).
2Installer sur un PC Windows avec un port COM série disponible (RS-232) ou un adaptateur USB-RS232.
3Durant l'installation, accepter l'installation des drivers série si demandée.
Info : Le manuel confirme (pub. 2098-UM001G-EN-P). La connexion au 2098-DSD-075 utilise le port série CN3 du drive via RS-232 ou RS-485.
A2
Préparer le câble de connexion PC → CN3
Câble RS-232 DB9 — connexion directe ou via adaptateur USB
1Identifier le connecteur CN3 sur le drive — connecteur DB-9 femelle sur la face supérieure du drive.
2Utiliser le câble série Allen-Bradley réf. 2090-UXUSBDC ou un câble RS-232 DB9 null-modem standard.
3Si le PC n'a pas de port RS-232 : utiliser un adaptateur USB → RS-232. Le chip FTDI est le plus fiable pour la stabilité de communication.
4Brancher une extrémité sur le PC (ou l'adaptateur) et l'autre sur CN3 du drive.
Attention : Intervention hors moteur. Le drive doit être sous tension avant de communiquer. De préférence, alimenter uniquement la carte de contrôle via les bornes L1-AUX et L2-AUX — dans ce cas la LED verte ne sera pas allumée. Si l'alimentation se fait uniquement depuis la carte de puissance, la LED sera allumée (non obligatoire).
Phase B — Connexion et accès au drive Paramétrage port série + détection du drive en ligne
B1
Configurer le port série dans Ultraware
Menu Tools → Serial Port — port COM + vitesse 38400 bps
1Ouvrir Ultraware. À la première ouverture, choisir Create new file puis cliquer OK.
2Aller dans le menu Tools → Serial Port.
3Dans la fenêtre "PC Communications Setup", configurer :

— Port COM : sélectionner le bon port (ex: COM3 — vérifier dans le Gestionnaire de périphériques Windows)
— Baud Rate : 38400 (valeur par défaut, confirmée par le manuel)
— Format : 8 bits de données, pas de parité
4Cliquer OK pour valider.
Important : Le manuel précise que les "Ultra3000 drives do not support the 57600 bps rate". Ne jamais utiliser 57600 — utiliser uniquement 38400.
Recommandation : Si le drive n'est pas détecté à 38400 bps, essayer le Baud Rate 9600 — certains drives configurés en usine ou après reset peuvent répondre à cette vitesse.
B2
Scanner le réseau pour détecter le drive
Scan automatique au démarrage — ou Tools → Rescan
1Ultraware scanne automatiquement les nœuds 0 à 255 au démarrage. Une barre de progression "Scan For On-Line Drives" s'affiche. Attendre la fin.
2Si le drive n'apparaît pas : aller dans Tools → Rescan pour relancer la recherche manuellement.
3Le drive détecté apparaît dans le panneau gauche "Workspace", sous la branche On-Line Drives. Son nom par défaut est Drive ou <Unnamed Drive>.
Connexion réussie si le drive apparaît sous "On-Line Drives". La LED Status du drive doit afficher un cercle tournant (version standard) ou un 4 fixe (version SERCOS).
Si le drive n'apparaît pas : vérifier le câble CN3, le port COM sélectionné, le baud rate (38400), et que la LED verte "Logic Power" est bien allumée sur le drive.
B3
Ouvrir la fenêtre de propriétés du drive
Double-clic sur le drive dans la branche On-Line Drives
1Dans le Workspace, double-cliquer sur le drive sous "On-Line Drives".
2La fenêtre "Drive Properties" s'ouvre dans la zone principale. Elle affiche en temps réel tous les paramètres du drive physique.
3Pour accéder aux sous-sections, cliquer sur les branches enfants dans le Workspace :
Motor
Modèle moteur sélectionné, direction, identification automatique
Tuning
Gains PID — Kp (position), Kv (vitesse), Ki (intégral)
Encoders
Type d'encodeur configuré, résolution, signaux utilisés
Digital Inputs
Affectation des 8 entrées numériques (enable, reset…)
Digital Outputs
Affectation des 4 sorties numériques + relais CN1-43/44
Faults
Historique des défauts enregistrés dans le drive
Info : Les paramètres sont lus directement depuis le drive online — pas besoin de les entrer manuellement. C'est la configuration réelle du drive physique.
Phase C — Lire, sauvegarder et exporter Backup de la configuration pour archivage ou remplacement
C1
Copier le drive online vers un fichier offline
La méthode la plus simple — Copy / Paste dans le Workspace
1Dans le Workspace, repérer le drive sous On-Line Drives.
2Faire un clic droit sur le drive → Copy (ou Ctrl+C).
3Cliquer sur le fichier Ultraware dans la branche File du Workspace (le fichier .udb créé au début).
4Faire clic droit → Paste (ou Ctrl+V). Le drive avec toute sa configuration apparaît maintenant dans la branche offline.
5Sauvegarder : File → Save (Ctrl+S). Donner un nom explicite, par exemple : Ultra3000-075_config_originale.udb.
Résultat : Le fichier .udb contient maintenant toute la configuration lue depuis le drive physique. C'est votre sauvegarde de référence. Elle peut être rechargée sur un drive de remplacement.
C2
Exporter en fichier .uxf (format portable)
File → Export — pour partage ou import sur un autre PC
1Cliquer sur le drive dans la branche offline (File).
2Menu File → Export (ou clic droit sur le drive → Export).
3Choisir un nom et un emplacement. Le fichier est sauvegardé avec l'extension .uxf (User data eXchange File).
4Ce fichier .uxf peut être importé sur n'importe quel PC avec Ultraware pour restaurer la configuration sur un drive de remplacement.
Différence : Le .uxf est le format de backup portable (un seul drive). Le .udb est la base de données Ultraware complète (peut contenir plusieurs drives). Les deux sont utiles — garder les deux.
C3
Paramètres de base importants à relever
Ce qu'il faut noter en priorité dans la configuration
1Motor Model — modèle exact du moteur configuré. Doit correspondre à la plaque signalétique du moteur physique.
2Auto Motor Iden — Enabled ou Disabled. Si Enabled : le drive lit les paramètres depuis l'encodeur intelligent (Stegmann) au démarrage.
3Operation Mode — mode de commande : Analog Velocity, Analog Position, Preset Velocity, etc. Important pour le câblage CN1.
4Drive Address — adresse série du drive (0 par défaut pour axe unique). Important si plusieurs drives en chaîne RS-485.
5Baud Rate — vitesse de communication configurée dans le drive. Doit être identique au paramètre PC (38400).
6Current Limits +/- — limites de courant positif et négatif. Protection moteur et drive.
7Branche Tuning : noter les gains Kp, Kv, Ki. Ce sont les paramètres les plus difficiles à retrouver si perdus — ils sont spécifiques à chaque application.
8Branche Faults : lire l'historique des défauts — utile pour comprendre l'état de santé du drive.
Conseil : En plus de la sauvegarde .udb, faire une capture d'écran de chaque fenêtre (Drive, Motor, Tuning, Encoders, Digital I/O). Double sécurité en cas de corruption du fichier.
C4
Remettre les paramètres aux valeurs d'usine (si nécessaire)
Commande "Reset EEPROM to Factory Setting" — à utiliser avec précaution
1Dans la fenêtre Drive Properties du drive online, cliquer Show Commands pour afficher la section commandes.
2Cliquer le bouton Reset EEPROM to Factory Setting.
3Cette commande réinitialise tous les paramètres aux valeurs d'usine. L'historique des défauts et le compteur "Time in Service" ne sont pas effacés.
Attention : N'effectuer cette opération qu'après avoir sauvegardé la configuration via C1 et C2. Une fois réinitialisé, la configuration personnalisée est définitivement perdue depuis le drive.
Phase D — Reprogrammer le drive pour un autre moteur Chargement d'un nouveau fichier de configuration moteur (.dxl)
D1
Importer le programme du nouveau moteur
Fichier au format .dxl — à importer depuis la branche Off-Line
1S'assurer de disposer du fichier de configuration moteur au format .dxl correspondant au nouveau moteur à piloter.
2Dans le Workspace, se positionner sur la branche Off-Line (section File).
3Faire un clic droit sur la branche Off-Line → sélectionner Importer le programme → sélectionner le fichier .dxl correspondant au nouveau moteur.
4Une fois importé, le profil du nouveau moteur apparaît dans la branche Off-Line. Copier ce profil (clic droit → Copy) et le coller dans la partie On-Line Drives (clic droit → Paste).
5Une fenêtre de confirmation s'ouvre — tout accepter pour valider le transfert de la nouvelle configuration vers le drive physique.
Attention : S'assurer que la communication série est bien établie avec le drive avant de lancer cette opération. Vérifier que le drive apparaît bien sous On-Line Drives. Une interruption de communication pendant le transfert peut corrompre la configuration du drive.
Après transfert : Aller dans Show Commands → cliquer Save to EEPROM pour sauvegarder définitivement la nouvelle configuration. Couper et remettre l'alimentation du drive pour valider.
